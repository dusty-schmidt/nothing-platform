#### What is the registration mechanism for agents with the hub?
Agents register with the hub by sending a POST request to the `/api/register` endpoint. The request must include a JSON body with an `agent_name` field. The hub will record the agent's name, the source IP address, and set its status to "online". This registration process allows the hub to track active agents and their network locations.

#### How do agents maintain heartbeat/status with the hub?
Agents maintain their status with the hub primarily through the registration endpoint. When an agent registers, the hub updates the agent's `last_seen` timestamp and `status` field in the `agent_sessions` database table. For continuous health checks, the hub provides a `/health` endpoint (e.g., `http://172.28.0.2:8080/health`) which returns a JSON response indicating the hub is "alive". Agents can periodically call this endpoint to verify the hub's availability. There is also a `/api/sync_status` endpoint that can be used to check synchronization status.
